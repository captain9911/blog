> 关键词：swift, 析构器, deinit

### 概述

* 只适用于类类型。
* 每个类最多有一个析构器。
* 析构器在实例被释放前会自动调用，不能手动调用。
* 如果子类实现了析构器，在析构结束的时候，会自动调用父类的析构器。
* 即使子类没有实现析构器，父类的析构器也会自动调用。
* 不带任何参数，也不需要()。

> 其实Swift的析构跟OC基本没差别，Swift的deinit相当于OC的dealloc。

### 析构器示例

```
deinit {
    // 执行析构过程
}
```
